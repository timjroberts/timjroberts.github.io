This is a quick post that demonstrates a function that will generate the _same_ UUID when presented with the _same_ seed value:

```typescript
const createUUIDfromSeed = (seed: string) => {
    const hash = crypto.createHash("sha256").update(seed).digest("hex");

    return [
      hash.substring(0, 8),
      hash.substring(8, 4),
      `4${hash.substring(12, 3)}`, // Set the version to 4
      `8${hash.substring(15, 3)}`, // Set the variant to 8 (RFC 4122)
      hash.substring(18, 12),
    ].join("-");
  }
```

With the version and variant values set, the UUID that is created should be compatible wherever a standard UUID/GUID is required.
